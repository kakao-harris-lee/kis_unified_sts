# 통합 투자 시스템 전략 설계서 (Master Investment System Design)

> **문서 목적**: 중장기 포트폴리오 투자와 자동매매(주식/선물)를 하나의 통합 체계로 운영하기 위한 마스터 설계서.
> 개별 전략 문서(VR 트레이딩 설계서, KOSPI200 선물 마이그레이션 지침서, 섹터 분석 리포트)의 상위 문서로서 자본 배분, 리스크 관리, 인프라 공유, 성과 피드백 루프를 정의한다.
> **작성일**: 2026-07-02
> **버전**: 1.0

---

## 0. 통합의 핵심 철학

세 개의 독립적인 투자 활동이 현재 각각 따로 운영되고 있다.

| 트랙 | 성격 | 시간 지평 | 현재 상태 |
|------|------|----------|----------|
| **A. 중장기 포트폴리오** | 방산·반도체 장비·로보틱스 섹터 투자 | 6개월~3년 | 종목 분석 완료, 배분 원칙 미정 |
| **B. 주식 자동매매** | VR+RSI+MA 일봉 신호 시스템 (KOSPI/KOSDAQ) | 수일~수주 스윙 | 설계서 완성, 실전 성과 개선 과제 존재 |
| **C. 선물 자동매매** | KOSPI200 선물, 뉴스+수급+규칙 기반 | 당일 청산 | RL → 하이브리드 전환 진행 중 |

통합의 목적은 세 트랙을 하나로 합치는 것이 아니라, **다음 4가지를 공유**하는 것이다.

1. **자본 배분 규칙** — 트랙 간 자금 이동의 명확한 기준
2. **리스크 예산(Risk Budget)** — 전체 자산 기준의 통합 MDD 한도
3. **시장 국면 판단(Market Regime)** — 하나의 매크로 신호가 세 트랙 모두에 영향
4. **데이터 인프라** — Redis Streams + ClickHouse를 전 트랙이 공유

핵심 원칙: **"트랙 B·C의 트레이딩은 트랙 A의 자산을 절대 침범하지 않는다."**

## 1. Layer 1 — 자본 구조 및 배분

### 1.1 3-계층 자본 구조

```
┌─────────────────────────────────────────────────────┐
│ 전체 투자 자산 (100%)                                │
├─────────────────────────────────────────────────────┤
│ Tier 1: 코어 포트폴리오        60~70%               │
│   └ 방산 / 반도체 장비 / 로보틱스 + 현금 버퍼        │
│                                                     │
│ Tier 2: 시스템 트레이딩 자본    20~30%               │
│   └ 주식 자동매매(B) + 선물 자동매매(C)              │
│                                                     │
│ Tier 3: 기회 자본 (예비)        10%                 │
│   └ 급락장 코어 추가 매수 전용, 평시 CMA/RP 대기     │
└─────────────────────────────────────────────────────┘
```

### 1.2 트랙 간 자금 이동 규칙 (일방향 원칙)

- **Tier 2 → Tier 1 (허용)**: 트레이딩 수익이 초기 자본의 +30%를 초과하면, 초과분의 50%를 코어 포트폴리오로 이전한다. 트레이딩 수익을 장기 자산으로 고정하는 장치다.
- **Tier 1 → Tier 2 (원칙 금지)**: 트레이딩 손실 보전을 위한 코어 자산 매도는 금지한다. 예외는 6개월 이상 트레이딩 시스템이 EV 양수를 검증한 후, 연 1회 리밸런싱 시점에만 허용한다.
- **Tier 3 발동 조건**: KOSPI 지수가 최근 고점 대비 -15% 이상 하락 & 코어 보유 종목의 투자 논거(수주잔고, CAPEX 사이클)가 훼손되지 않았을 때만 분할 투입(3회 분할)한다.

### 1.3 선물 계좌 격리

트랙 C(선물)는 원금 초과 손실이 가능하므로 **별도 계좌로 물리적 격리**하고, Tier 2 자본 중에서도 상한을 둔다.

```
Tier 2 내부 배분:
├─ 주식 자동매매(B): Tier 2의 70%
└─ 선물 자동매매(C): Tier 2의 30% (전체 자산의 약 6~9%)
    └─ 월 1회만 증거금 보충 가능, 한도 초과 손실 시 당월 중단
```

## 2. Layer 2 — 트랙 A: 중장기 코어 포트폴리오

### 2.1 섹터 배분 프레임

기존 분석(방산/반도체 장비/로보틱스)의 섹터 성격 차이를 배분 비중에 직접 반영한다.

| 섹터 | 실적 확실성 | 역할 | 기본 비중 (코어 내) |
|------|-----------|------|-------------------|
| 방산 | 수주잔고 확정형 | 앵커(안정 성장) | 35% |
| 반도체 장비 | CAPEX 사이클 연동 | 사이클 알파 | 35% |
| 로보틱스 | 기대 선반영형 | 성장 옵션 | 15% |
| 현금 버퍼 | — | 리밸런싱 재원 | 15% |

### 2.2 종목 편입/편출 규칙

**편입 기준 (전부 충족 시):**
1. 투자 논거를 한 문장으로 기술 가능 (예: "TC 본더 독점 → HBM4 양산 수혜")
2. 논거를 무효화하는 조건(Kill Criteria)을 사전에 명시
3. 분할 매수 계획 수립 (최소 3회 분할, 1회 매수는 목표 비중의 40% 이하)

**편출 기준 (하나라도 발동 시 검토):**
- Kill Criteria 발동 (예: 로보틱스 IPO 재추진 → HD현대 할인 논거 소멸)
- 섹터 비중이 목표 대비 ±10%p 이탈 (리밸런싱)
- 더 나은 논거의 종목 발견 시 교체 (단, 월 1회 이상 교체 금지 — 회전율 억제)

### 2.3 리밸런싱 주기

- **정기**: 분기 1회 (1/4/7/10월 첫 거래주)
- **수시**: 단일 종목이 코어의 25%를 초과하거나, 섹터 비중이 ±10%p 이탈 시
- **금지**: 일간 가격 변동에 따른 임의 매매 (트레이딩 충동은 Tier 2에서만 해소)

## 3. Layer 3 — 트랙 B: 주식 자동매매 (VR 시스템)

기존 VR 설계서를 유지하되, 실전에서 확인된 4대 문제에 대한 해결 원칙을 상위 규칙으로 확정한다.

### 3.1 4대 문제 → 확정 규칙

| 문제 | 확정 규칙 |
|------|----------|
| 진입 타이밍 지연 | VR을 **1차 트리거**, RSI/MA는 **필터 전용**으로 역할 분리. 일봉 신호 → 익일 시가 또는 눌림목 진입만 허용 |
| 신호 과다/과소 | **시장 국면 필터** (Layer 5) 통과 신호만 실행. 종목 신호와 지수 방향 불일치 시 자동 기각 |
| 백테스트-실전 괴리 | 백테스트에 슬리피지 0.3% + 수수료/세금 반영 필수. 실체결 데이터를 ClickHouse에 적재해 **주간 슬리피지 리포트** 자동 생성 |
| 손절/익절 미정의 | 아래 3.2를 시스템 하드코딩 규칙으로 채택 |

### 3.2 손절/익절 규격 (확정안)

```
진입 시점에 3개 가격을 동시 확정 (진입 후 변경 금지):

├─ 손절가: 진입가 - 2.0 × ATR(14)    [최대 손실 -7% 캡]
├─ 1차 익절: 진입가 + 2.0 × ATR(14)  [보유량 50% 청산]
└─ 트레일링: 1차 익절 후 잔량은 10일 최저가 이탈 시 청산

포지션 사이징:
└─ 1회 리스크 = 트랙 B 자본의 1.0%
   (수량 = 리스크 금액 ÷ (진입가 - 손절가))
```

### 3.3 신호 생성 흐름 (기존 설계서와의 관계)

```
[VR 설계서의 신호 엔진] → [Layer 5 시장 국면 필터] → [3.2 리스크 규격 적용] → 주문
```

기존 설계서의 신뢰도(confidence)·경고(warnings) 체계는 그대로 유지하며, 국면 필터가 그 앞단에 추가된다.

## 4. Layer 4 — 트랙 C: KOSPI200 선물 자동매매

기존 마이그레이션 지침서(뉴스+수급+규칙 기반)의 핵심을 승계하고, 통합 체계에서의 위치를 정의한다.

### 4.1 3-Tier 신호 체계 (승계)

- **Tier 1 (장전)**: 미 선물 야간 등락, 환율, 매크로 뉴스 감성 → 당일 방향 가설
- **Tier 2 (장 초반)**: 외국인 선물 누적 순매수, 현선물 괴리 → 진입 방향 확정
- **Tier 3 (필터)**: 1분봉 RSI, ATR 변동성 상태, VWAP 기준 → 진입 타이밍

### 4.2 생존 규칙 (승계 + 통합 강화)

```
├─ 1회 최대 리스크: 트랙 C 자본의 1.5%
├─ 일일 손실 한도: 3% → 당일 매매 중단
├─ 주간 손실 한도: 7% → 시스템 재검증 (실매매 정지, 페이퍼 트레이딩 전환)
├─ 연속 4패: 2주간 포지션 사이즈 50% 축소
└─ RL 모델: 메인 로직 금지, 진입 적합 확률 필터로만 보조 사용
```

### 4.3 트랙 C의 통합 체계 내 특별 규칙

- 트랙 C 월간 손실이 트랙 C 자본의 15% 도달 시 → **당월 완전 중단** + 원인 분석 리포트 작성 후 재개
- 트랙 C 수익은 분기마다 초기 증거금 수준으로 리셋하고, 초과분은 Tier 2 주식 계좌로 인출 (선물 계좌 비대화 방지)

## 5. Layer 5 — 통합 시장 국면 엔진 (Market Regime Engine)

세 트랙이 공유하는 단일 매크로 판단 모듈. **이 레이어가 통합 시스템의 심장이다.**

### 5.1 국면 정의 (일봉 기준, 매일 장 마감 후 산출)

| 입력 지표 | 산출 |
|----------|------|
| KOSPI/KOSDAQ 지수 VR | 시장 과열/침체 |
| 지수 MA 배열 (5/20/60) | 추세 방향 |
| 지수 20일 변동성 (ATR%) | 변동성 상태 |
| 외국인 20일 누적 순매수 | 수급 방향 |

→ 4개 입력을 조합해 국면을 3가지로 분류: **RISK_ON / NEUTRAL / RISK_OFF**

### 5.2 국면별 트랙 반응 매트릭스

| | RISK_ON | NEUTRAL | RISK_OFF |
|--|---------|---------|----------|
| **트랙 A (코어)** | 정상 보유 | 정상 보유 | 신규 매수 중단, Tier 3 발동 감시 |
| **트랙 B (주식)** | 매수 신호 정상 실행 | 신뢰도 HIGH 신호만 실행 | 신규 진입 전면 금지, 보유분 손절 규칙만 가동 |
| **트랙 C (선물)** | 롱/숏 양방향 | 양방향 (사이즈 70%) | 숏 편향 허용, 사이즈 50% |

### 5.3 구현 위치

```
Redis Streams:
└─ stream:regime.daily  ← 국면 엔진이 매일 1회 발행
     ├─ 트랙 B 신호 엔진이 구독 (필터)
     ├─ 트랙 C Decision Engine이 구독 (사이즈 조절)
     └─ 트랙 A는 주간 요약 리포트로 확인 (자동 매매 아님)
```

## 6. Layer 6 — 데이터 및 인프라 통합

기존 인프라(Redis Streams, ClickHouse)를 전 트랙의 공용 기반으로 확장한다.

### 6.1 통합 스트림 구조

```
Redis Streams (Event Bus)
├─ stream:market.daily      ← 일봉 OHLCV (트랙 A/B용)
├─ stream:market.tick       ← 실시간 체결 (트랙 C용, 기존)
├─ stream:news.scored       ← 뉴스 감성 (트랙 C 주도, 트랙 A 참고)
├─ stream:foreign.flow      ← 외국인 수급 (트랙 C + 국면 엔진)
├─ stream:regime.daily      ← 시장 국면 (신규, 전 트랙 공유)
└─ stream:signal.final      ← 트랙별 최종 신호 (trackId 태깅)
```

### 6.2 ClickHouse 통합 스키마 원칙

- 모든 신호·주문·체결·슬리피지를 **트랙 ID로 태깅**하여 단일 테이블 체계에 적재
- 트랙 A의 매수/매도도 수동이지만 동일 스키마로 기록 (통합 성과 분석을 위해)
- 주간 배치: 트랙별 Edge 검증 + 통합 자산 곡선(Equity Curve) 산출

### 6.3 환경 분리 연계

개인/회사 Claude Code 환경 분리(`CLAUDE_CONFIG_DIR` 방식)와 동일한 원칙으로, 본 시스템은 **개인 환경(`~/.claude-personal/`)에서만 개발·운영**한다. 원격 서버 적용 시에도 동일한 분리 구조를 유지한다.

## 7. Layer 7 — 통합 리스크 예산 및 서킷 브레이커

### 7.1 전체 자산 기준 통합 한도

```
통합 MDD 감시 (월간, 전체 자산 기준):
├─ -5% 도달: 트랙 B/C 신규 진입 사이즈 50% 축소
├─ -8% 도달: 트랙 B/C 신규 진입 전면 중단 (보유분 규칙 청산만)
└─ -12% 도달: 전 시스템 정지 + 월간 리뷰 문서 작성 후 재개
    (트랙 A는 매도하지 않음 — 코어는 MDD 트리거 대상 아님)
```

### 7.2 상관관계 리스크

트랙 A(반도체 장비 비중 高)와 트랙 B(KOSPI/KOSDAQ 종목)는 시장 하락 시 동반 손실 가능성이 크다. 따라서:

- 트랙 B는 트랙 A 보유 종목과 **동일 종목 중복 진입 금지**
- 트랙 B 진입 종목 중 반도체 섹터 비중 상한: 트랙 B 포지션의 40%

---

## 8. Layer 8 — 성과 측정 및 피드백 루프

### 8.1 주기별 리뷰 체계

| 주기 | 내용 | 산출물 |
|------|------|--------|
| 주간 | 트랙 B/C 슬리피지·승률·Edge 검증 (ClickHouse 배치) | 자동 리포트 |
| 월간 | 통합 자산 곡선, 트랙별 기여도, MDD 점검 | 월간 요약 1페이지 |
| 분기 | 트랙 A 리밸런싱 + Kill Criteria 점검, 트랙 간 자금 이동 실행 | 리밸런싱 기록 |
| 연간 | 전략 구조 자체의 재검토 (배분 비율, 트랙 존폐) | 설계서 버전 업 |

### 8.2 트랙별 성공/중단 판정 기준

- **트랙 B**: 6개월 롤링 기준 실전 수익률이 (백테스트 기대치 - 슬리피지)의 60% 이상 → 유지. 미달 2분기 연속 → 페이퍼 트레이딩 강등
- **트랙 C**: 첫 3개월 목표는 본전 유지. 6개월 후 EV 양수 미확인 시 자본 50% 축소, 12개월 미확인 시 트랙 폐지 검토
- **트랙 A**: 벤치마크(KOSPI TR) 대비 연간 성과로 평가하되, 3년 미만 구간에서는 판정 유보
